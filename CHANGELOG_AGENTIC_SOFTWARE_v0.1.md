# Changelog — Agentic Software Systems v0.1

## Agentic Software Systems Domain Profile v0.1

* Added environment-neutral Agentic Software Systems Domain Profile.
* Added GitHub-style agentic workflow adoption guide.
* Added agentic software governance mapping.
* Added agentic software risk-benefit matrix.
* Added agentic software adoption checklist.
* Added illustrative agentic pull request Legitimacy Binding Artifact example.
* Defined agentic software concepts including repository actions, pull requests, CI/CD, deployment workflows, tool calls, and review responsibilities.
* Added risk categories for low-risk assistive support, medium-risk repository support, high-risk workflow or security use, high-risk operational use, and prohibited or exceptional use.
* Added software environment adaptation note for translating profile terms across local platforms, workflows, roles, permissions, and deployment environments.
* Clarified that the profile does not create software safety, cybersecurity, legal, regulatory, compliance, audit, certification, or production-readiness approval by itself.
* Preserved the Governance Constitution's core requirements for authorization, scope, expiry, evidence references, non-delegation, proof-carrying change, reviewability, and accountable responsibility.
